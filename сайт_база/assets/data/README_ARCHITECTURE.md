# Архитектура сайта-портфолио

Этот документ описывает структуру и ключевые принципы работы сайта. Он предназначен для быстрого введения в проект нового разработчика или AI-ассистента.

## 1. Обзор сайта

Это **статический сайт-портфолио** для художницы Наташи Пастуховой. Его главная цель — демонстрация её работ.

- **Технологии**: Сайт создан с использованием чистого HTML, CSS и JavaScript. В нём нет фреймворков, библиотек или систем сборки (типа Webpack).
- **Контент**: Данные о проектах (названия, описания, изображения) хранятся централизованно в JSON-файле, что делает управление контентом удобным.

## 2. Структура файлов

Проект имеет простую и понятную структуру:

```
/
├── index.html              # Главная страница, отображает сетку всех проектов.
├── about.html              # Страница "О Наташе" с её CV.
├── README_ARCHITECTURE.md  # Этот файл.
│
├── css/
│   └── style.css           # Единственный файл стилей для всего сайта.
│
├── js/
│   └── main.js             # Единственный JS-файл, содержащий всю логику сайта.
│
├── assets/
│   ├── data/
│   │   └── projects.json   # Ключевой файл: содержит данные обо всех проектах.
│   └── images/
│       └── ...             # Изображения для проектов, разложенные по папкам.
│
└── projects/
    └── project-1/
        └── index.html      # Пример HTML-страницы для отдельного проекта.
```

**Важно**: Файлы `projects.html` и `js/script.js` являются устаревшими и должны быть удалены из проекта, чтобы избежать путаницы.

## 3. Логика ключевых частей

### Шапка сайта (Header/Navbar)

Шапка не является переиспользуемым компонентом. Это статический HTML-блок, который скопирован в каждый HTML-файл (`index.html`, `about.html` и т.д.). Навигация реализована через стандартные теги `<a>`.

### Сетка проектов (Главная страница)

Сетка карточек на `index.html` генерируется **динамически**.

1.  Скрипт `js/main.js` при загрузке страницы отправляет запрос (`fetch`) к файлу `assets/data/projects.json`.
2.  Получив данные, он создаёт HTML-разметку для каждой карточки проекта.
3.  Готовая разметка вставляется в `<div id="cards-start">`.
4.  Этот же скрипт управляет фильтрацией проектов по категориям («All», «Murals», «Sculptures»).

### Страница проекта и слайдер

Страницы отдельных проектов (например, `projects/project-1/index.html`) на данный момент являются **статическими HTML-файлами**.

- **Слайдер**: Логика слайдера находится в `js/main.js`. Он управляет переключением слайдов (стрелки, точки, свайпы), счётчиком и открытием изображений в полноэкранном режиме (лайтбокс).
- **Контент**: Текст (заголовок, описание, кредиты) и изображения для слайдера пока что жёстко прописаны в HTML-коде каждой страницы проекта. В будущем это можно будет также сделать динамическим, подгружая данные из `projects.json`.

## 4. Как добавить новый проект

Процесс состоит из трёх шагов:

1.  **Добавить данные в `projects.json`**:
    - Откройте `assets/data/projects.json`.
    - Скопируйте существующий объект проекта и вставьте его как новый элемент массива.
    - Заполните все поля: `id`, `name`, `desc`, `cover` (путь к обложке), `credits`, `images` (список всех фото проекта) и `url` (ссылка на будущую папку проекта, например `/projects/new-project/`).

2.  **Создать HTML-страницу проекта**:
    - Создайте новую папку в директории `projects/` (например, `new-project`).
    - Скопируйте `projects/project-1/index.html` в эту новую папку.
    - В скопированном файле вручную замените тексты и пути к изображениям в слайдере на данные нового проекта.

3.  **Загрузить изображения**:
    - Создайте соответствующую папку для изображений в `assets/images/projects/`.
    - Загрузите туда все файлы изображений для нового проекта.

## 5. Запуск и разработка

- **Зависимости**: У проекта **нет внешних зависимостей**.
- **Запуск**: Поскольку сайт использует `fetch` для загрузки `projects.json`, его нужно открывать через локальный веб-сервер. Самый простой способ — использовать расширение **Live Server** для VS Code. Открытие `index.html` напрямую через `file://` может привести к ошибкам загрузки данных.
