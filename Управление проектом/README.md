- `01_бриф.md` - зачем сайт, для кого, какой образ и что считаем готовой первой версией.
- `02_план.md` - где мы сейчас, куда идем, бэклог и ближайшие решения.
- `03_контент.md` - проекты, фото, CV, магазин, кураторский текст, переводы.
- `04_техника.md` - архитектура сайта, обновление, скорость, изображения, видео, текущий аудит.
- `05_редакция.md` - будущая команда, роли, brief'ы для агентов и правила запуска.

## Правило проекта

Не запускаем агентов и не уходим в дизайн/разработку, пока не согласованы бриф, архитектура, список проектов и входные материалы.

## Git

Git нужен, но подключать его лучше не как формальность, а перед началом активной разработки.

Рекомендация:

1. Сначала держим порядок в папках и управленческих документах.
2. Перед первыми крупными правками сайта и контента инициализируем Git.
3. Делаем понятные коммиты по этапам: `project setup`, `content structure`, `homepage`, `project pages`, `about`, `shop`, `qa fixes`.

Зачем он нужен:

- видеть историю изменений;
- безопасно откатывать неудачные решения;
- разделять работу по агентам/ролям;
- проще публиковать на Netlify или другом хостинге.

Почему не прямо в эту секунду:

- сначала нужно закончить первичную организацию проекта;
- в папке есть много черновиков и устаревших файлов;
- Git лучше запускать перед осознанной первой базовой фиксацией.
