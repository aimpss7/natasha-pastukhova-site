# Управление проектом

Этот раздел - штаб проекта сайта-портфолио Наташи Пастуховой.

## Как читать папку

- `01_бриф.md` - зачем сайт, для кого, какой образ и что считаем готовой первой версией.
- `02_план.md` - где мы сейчас, куда идем, бэклог и ближайшие решения.
- `03_контент.md` - проекты, фото, CV, магазин, кураторский текст, переводы.
- `04_техника.md` - архитектура сайта, обновление, скорость, изображения, видео, текущий аудит.
- `05_редакция.md` - будущая команда, роли, brief'ы для агентов и правила запуска.

## Структура папок

- `site/` - текущий статический сайт и web-ready ассеты.
- `materials/docs/` - исходные документы, CSV из Notion и мастер-таблица проектов.
- `materials/source-photos/` - тяжелые оригиналы фото и видео, не хранятся в Git.
- `materials/references/` - локальные сохраненные сайты-референсы, не хранятся в Git.
- `Управление проектом/` - штаб проекта, план, контент, техника и роли.

## Правило проекта

Не запускаем агентов и не уходим в дизайн/разработку, пока не согласованы бриф, архитектура, список проектов и входные материалы.

## Git

Git включен для проекта и используется как журнал решений и правок.

Первая фиксация: `b2f4f92 Initial project baseline`.

Текущее правило:

1. В историю кладем сайт, управленческие документы, CSV/PDF-источники и web-ready изображения.
2. Не кладем тяжелые оригиналы из `materials/source-photos/` и локальные референсы `materials/references/`.
3. Делаем понятные коммиты по этапам: `project setup`, `content structure`, `homepage`, `project pages`, `about`, `shop`, `qa fixes`.

Зачем он нужен:

- видеть историю изменений;
- безопасно откатывать неудачные решения;
- разделять работу по агентам/ролям;
- проще публиковать на Netlify или другом хостинге.

Почему это важно сейчас:

- уже начались правки сайта и контента;
- нужно безопасно проверять гипотезы и откатывать неудачные решения;
- дальше можно давать агентам задачи без потери контроля над изменениями.
